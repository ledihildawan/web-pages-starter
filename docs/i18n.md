# i18n System

Build multilingual applications with 42 locales, automatic RTL, native digits, and locale-specific formatting.

---

## Quick Start

```njk
{{ i18n.t('page.title') }}                             <!-- Translation -->
{{ i18n.t('page.welcome', {name: 'John'}) }}           <!-- With variables -->
{{ i18n.formatNumber(1234.56) }}                       <!-- 1,234.56 -->
{{ i18n.formatCurrency(99.99, 'USD') }}              <!-- $99.99 -->
{{ i18n.formatLocalPrice(plan) }}                     <!-- Auto regional pricing -->
{{ i18n.formatDate(new Date()) }}                      <!-- 6/3/26 -->
{{ i18n.formatRelativeTime(-1, {unit: 'day'}) }}      <!-- 1 day ago -->
```

No `| safe` needed. All functions output HTML with data attributes for runtime language switching.

### Three Integration Layers

| Layer | Location | Purpose |
|-------|----------|---------|
| **Build Time** | `rsbuild.config.ts:202` | Nunjucks renders with `autoescape: false` |
| **Client Load** | `src/scripts/lib/i18n.ts:309` | i18next with `interpolation: { escapeValue: false }` |
| **Runtime** | `src/scripts/lib/i18n.ts:375` | Direct `innerHTML` updates on language switch |

### BCP 47 Compliance

All locale codes in this system follow BCP 47 standard:
- Format: `language-script-region` or `language-region`
- Language subtags from ISO 639 (e.g., `zh`, `en`, `id`)
- Script subtags from ISO 15924 (e.g., `Hans`, `Hant`, `Latn`)
- Region subtags from ISO 3166 (e.g., `CN`, `US`, `ID`)

### Inline Hydration Pattern

All i18n functions output `<span>` elements with data attributes:

```njk
{{ i18n.t('page.title') }}
<!-- Output: <span data-i18n="page:title">Welcome</span> -->

{{ i18n.formatNumber(1234.56) }}
<!-- Output: <span data-format-number="1234.56">1,234.56</span> -->
```

At runtime, when BCP 47 locale changes:
1. `translatePage()` updates all `[data-i18n]` elements
2. `updateFormattedElements()` updates all `[data-format-*]` elements
3. No page refresh required

## Translation API

| Function | Params | Returns | Use For |
|----------|--------|---------|---------|
| `t(key, vars, options?)` | key, vars = {}, {raw?, native?, universal?, className?} | `<span data-i18n="key">` | Text with variables |
| `html(key, vars)` | key, vars = {} | `<span data-i18n-html="key">` | Rich HTML content |
| `text(key, vars)` | key, vars = {} | Plain text | Attributes (alt, placeholder) |
| `attr(key, attrName, vars)` | key, attrName, vars = {} | `attrName="value"` | HTML attributes |
| `plural(key, count, vars, options?)` | key, count, vars = {}, {raw?, className?} | `<span data-i18n-plural="key">` | Pluralized text |

### Options Reference

| Option | Type | Effect |
|--------|------|--------|
| `raw` | boolean | Return plain text, no wrapper |
| `native` | boolean | Force native digits (٠١٢٣٤) |
| `universal` | boolean | Force universal digits (0123456789) |
| `className` | string | Add custom class to wrapper |

### Locale File Patterns

### Alpine.js Store Integration

```njk
<div x-data>
  <template x-for="lang in $store.i18n.languages" :key="lang.code">
    <button @click="$store.i18n.change(lang.code)">
      <span x-text="lang.flag"></span>
      <span x-text="lang.label"></span>
    </button>
  </template>
</div>
```

**Note:** `lang.code` is the BCP 47 locale code (e.g., `'en-US'`, `'id-ID'`, `'ar-SA'`)

### Store API

| Property | Type | Description |
|----------|------|-------------|
| `$store.i18n.languages` | Array<{code, label, flag}> | All available BCP 47 locales |
| `$store.i18n.current` | string | Current BCP 47 locale code |
| `$store.i18n.change(localeCode)` | function | Switch to new BCP 47 locale |

### What Happens On Switch

1. `localStorage.setItem('i18nextLocale', localeCode)` - Store BCP 47 locale
2. `<html lang="localeCode" dir="ltr|rtl" data-script="writingSystem">` - Update HTML attributes
3. All `[data-i18n]` elements update with new translations
4. All `[data-format-*]` elements update with new locale formatting
5. No page refresh required

**Example:** Switching to Arabic (`ar-SA`)
```html
<html lang="ar-SA" dir="rtl" data-script="arabic">
```

**Example:** Switching to Japanese (`ja-JP`)
```html
<html lang="ja-JP" dir="ltr" data-script="japan">
```

## Locale Configuration

### BCP 47 Language Tags

All locale codes follow **BCP 47** standard format: `language-script-region` or `language-region`

| Component | Description | Examples |
|-----------|-------------|----------|
| **Language subtag** | ISO 639-1/639-2 code (2-3 letters) | `zh`, `en`, `id`, `ar`, `ja` |
| **Script subtag** | ISO 15924 code (4 letters, optional) | `Hans` (Simplified), `Hant` (Traditional), `Latn` (Latin), `Arab` (Arabic) |
| **Region subtag** | ISO 3166-1 code (2 letters) | `CN`, `US`, `ID`, `SA`, `JP` |

**Examples:**
- `zh-Hans-CN` = Chinese (Simplified script, China region)
- `zh-CN` = Chinese (default script, China region)
- `en-US` = English (United States)
- `ar-SA` = Arabic (Saudi Arabia)

### Supported Locales (42)

| Geographic Region | BCP 47 Locale Codes | Special Features |
|-------------------|---------------------|------------------|
| **Southeast Asia** | id, id-ID, th-TH, vi-VN | Thai native digits |
| **East Asia** | zh-CN, zh-TW, zh-HK, zh-SG, ja-JP, ko-KR | CJK numbers |
| **South Asia** | hi-IN, hi-NP | Devanagari native digits |
| **Middle East** | ar, ar-SA, ar-AE, ar-EG, ar-MA, ar-TN | RTL + Arabic digits |
| **Europe** | en-GB, de-DE, de-AT, de-CH, fr-FR, fr-BE, fr-CH, fr-CA, es-ES, es-MX, es-AR, es-CO, es-PE, pt-PT, pt-BR, pt-AO, pt-MZ, nl-BE, nl-NL, it-IT | - |
| **Americas** | en-US, en-CA, en-AU, en-NZ, en-ZA, en-IN | - |
| **Africa/Russia** | ru-RU | - |

### Numbering Systems (27)

| System | Digits | Locales |
|--------|--------|---------|
| latn | 0123456789 | Most locales |
| arab | ٠١٢٣٤٥٦٧٨٩ | Arabic |
| thai | ๐๑๒๓๔๕๖๗๘๙ | Thai |
| deva | ०१२३४५६७८९ | Hindi |
| hanidec | 〇一二三四五六七八九 | Chinese/Japanese |

### BCP 47 Locale Fallback Chain

Fallback resolves by **language subtag** then **region subtag**:

```
zh-SG → zh-CN → en-US     # Singapore Chinese → Simplified Chinese → Default
zh-HK → zh-TW → en-US     # Hong Kong Chinese → Traditional Chinese → Default
es-AR → es-ES → en-US     # Argentine Spanish → Spain Spanish → Default
pt-AO → pt-PT → en-US     # Angolan Portuguese → Portugal Portuguese → Default
```

### Font Loading Strategy

**Automatic by Writing System:**

When a BCP 47 locale is selected, the system:
1. Detects the writing system from the locale's numbering system
2. Sets `data-script` attribute on `<html>` element
3. CSS loads appropriate fonts for that writing system

**Example:**
```html
<!-- Arabic (ar-SA) -->
<html lang="ar-SA" dir="rtl" data-script="arabic">
<!-- Loads: Noto Sans Arabic -->

<!-- Japanese (ja-JP) -->
<html lang="ja-JP" dir="ltr" data-script="japan">
<!-- Loads: Noto Sans CJK JP (Variable) -->

<!-- Hindi (hi-IN) -->
<html lang="hi-IN" dir="ltr" data-script="devanagari">
<!-- Loads: Noto Sans Devanagari -->
```

### Writing System Detection

The system uses **NUMBERING_SYSTEM_TO_WRITING_SYSTEM** mapping from `src/configs/locales.ts`:

```typescript
// Numbering System → Writing System
'latn' → 'latin'
'arab' → 'arabic'
'thai' → 'thai'
'deva' → 'devanagari'
'hanidec' → 'cjk'
'cyrl' → 'cyrillic'
// ... and more
```

This enables automatic font selection based on the locale's native numbering system.

## RTL Layouts

### What Happens Automatically

1. `<html dir="rtl">` added
2. `is-rtl` class added
3. Layout mirrors with logical CSS

### Write RTL-Friendly CSS

```css
/* ✅ Logical properties (LTR + RTL) */
.sidebar { margin-inline-start: 1rem; }
.content { padding-inline-end: 2rem; }
.header { border-inline-end: 1px solid #ccc; }

/* ❌ Physical properties (breaks in RTL) */
.sidebar { margin-left: 1rem; }
.content { padding-right: 2rem; }
.header { border-right: 1px solid #ccc; }
```

### Logical Property Mapping

| Logical | LTR | RTL |
|---------|-----|-----|
| `margin-inline-start` | `margin-left` | `margin-right` |
| `margin-inline-end` | `margin-right` | `margin-left` |
| `padding-inline-start` | `padding-left` | `padding-right` |
| `border-inline-start` | `border-left` | `border-right` |

## Best Practices

### Do This

```njk
<!-- All user-facing text through i18n -->
{{ i18n.t('page.hero.title') }}

<!-- Use formatters for numbers -->
{{ i18n.formatCurrency(price, 'USD') }}
{{ i18n.formatNumber(metricValue) }}

<!-- Logical CSS for layout -->
.sidebar { margin-inline-start: 1rem; }

<!-- Unicode or HTML entities work -->
"copyright": "© 2026"
"copyright": "&copy; 2026"
```

### Avoid This

```njk
<!-- Never hardcode text -->
<h1>Welcome</h1>

<!-- Don't skip formatters -->
{{ price }}

<!-- Don't use physical CSS -->
.sidebar { margin-left: 1rem; }
```

## Troubleshooting

| Problem | Cause | Solution |
|---------|-------|----------|
| `[page.key]` appears | Key missing from locale | Add key to all 42 locale files |
| Won't update on switch | Missing data attributes | Hard refresh, check console |
| Wrong currency format | Not using formatter | Use `formatCurrency()` |
| RTL layout broken | Physical CSS | Use `margin-inline-start` |
| Native digits not showing | Not using format function | Use format functions |
| Pluralization broken | Missing suffix | Add `_one`, `_other` to keys |
| Build fails | Invalid JSON5 | Check JSON5 syntax |
