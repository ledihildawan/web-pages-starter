# i18n System

Complete internationalization system for 12 languages with automatic font switching, RTL support, and locale-specific data.

---

## Quick Start

### Add Translation
```bash
# 1. Add to ALL 12 locale files
src/locales/{lang}/{page}.json5

# 2. Generate types
bun run gen:i18n
```

### Add Language
```bash
# 1. Add to src/configs/locales.ts
# 2. bun run sync:locales  # Create locale files
# 3. Translate files
# 4. bun run fetch:rates    # Update exchange rates
# 5. bun run gen:i18n       # Generate types
# 6. bun add @fontsource-variable/noto-sans-{variant}
```

## Architecture

```
┌─────────────────┐
│   Config        │  locales.ts (languages, fonts, currency)
└────────┬────────┘
         │
┌────────▼────────┐
│   Content       │  locale files (translation data)
└────────┬────────┘
         │
┌────────▼────────┐

└────────┬────────┘
         │
┌────────▼────────┐
│   Present       │  HTML with fonts & RTL styles
└─────────────────┘
```

### Key Resolution

| Prefix | File | Example |
|--------|------|---------|
| `page.key` | `{page_id}.json5` | `page.hero.title` → `contact.json5:hero.title` |
| `comp.name.key` | `components/{name}.json5` | `comp.cta.heading` → `components/cta.json5:heading` |
| `common.key` | `common.json5` | `common.site_name` → `common.json5:site_name` |

**Page ID mapping:** `src/pages/{page}/index.json5` → `{ "page_id": "contact" }`

## API Reference

### Translation Macros

| Macro | Purpose |
|-------|---------|
| `i18n.t('key')` | Plain text (auto native numbers) |
| `i18n.tNative('key')` | Force native digits (٠١٢٣٤) |
| `i18n.tUniversal('key')` | Force universal digits (0-9) |
| `i18n.html('key')` | HTML content (dynamic) |
| `i18n.attr('key', 'placeholder')` | Attribute translation |
| `i18n.plural('key', count)` | Pluralization |
| `i18n.v('key')` | Get value (no markup) |

**With variables:**
```njk
{{ i18n.t('page.welcome', {name: 'John'}) }}
// JSON5: "Welcome, {{name}}!"
```

### Formatting Macros

| Category | Macro | Output Example |
|----------|-------|----------------|
| **Numbers** | `formatNumber(val)` | `1234.56` → `1,234.56` |
| | `formatPercent(val)` | `0.99` → `99%` |
| | `formatBytes(val)` | `1024` → `1 KB` |
| | `formatScientific(val)` | `1230` → `1.23E+3` |
| | `formatAbbreviated(val)` | `1500000` → `1.5M` |
| | `formatOrdinal(val)` | `1` → `1st` |
| | `formatCardinal(val)` | `1` → `satu` (ID) |
| **Currency** | `formatCurrency(val)` | `99.99` → `$99.99` |
| | `convertCurrency(val)` | `99 USD` → `Rp1.777.800` |
| | `formatLocalPrice(plan)` | Regional pricing |
| | `formatLocalPriceDiscounted(plan, 0.8)` | 20% off |
| **Date/Time** | `formatDate(val)` | `'2026-05-28'` → `5/28/26` |
| | `formatTime(val)` | → `1:30 PM` |
| | `formatDateTime(val)` | → `5/28/26, 1:30 PM` |
| | `formatDuration(val)` | `3661` → `1 hour 1 minute` |

| Code | Language | Font | Currency | RTL | Native Numbers |
|------|----------|-------|----------|-----|----------------|
| `id-ID` | Bahasa Indonesia | Inter | IDR | - | - |
| `en-US` | English (US) | Inter | USD | - | - |
| `ja-JP` | 日本語 | Noto Sans JP | JPY | - | - |
| `zh-CN` | 简体中文 | Noto Sans SC | CNY | - | - |
| `ar-SA` | العربية | Noto Sans Arabic | SAR | ✅ | ✅ |
| `es-ES` | Español | Inter | EUR | - | - |
| `pt-BR` | Português (Brasil) | Inter | BRL | - | - |
| `hi-IN` | हिन्दी | Noto Sans Variable | INR | - | ✅ |
| `ko-KR` | 한국어 | Noto Sans KR | KRW | - | - |

| `ru-RU` | Русский | Inter | RUB | - | - |

**Native numbers:** Arabic (٠-٩) and Hindi (०-९) auto-convert. Use `tNative()` to force for any language.

### Fallback Chains

For locale variants not directly supported (e.g., zh-SG, en-GB):

```
zh-SG → zh-CN → id-ID
en-GB → en-US → id-ID
es-MX → es-ES → id-ID
pt-PT → pt-BR → id-ID
```

**When used:** Browser sends `Accept-Language: zh-SG` → falls back to `zh-CN` instead of `id-ID`.

**Implementation:** `src/scripts/utils/locale.ts`

## Advanced Features

### Pluralization

| Language | Forms | Keys |
|----------|-------|------|
| id-ID, ja-JP, ko-KR, zh-CN | 1 | `_other` |
| en-US, es-ES, pt-BR, fr-FR, de-DE, hi-IN | 2 | `_one`, `_other` |
| ar-SA | 6 | `_zero`, `_one`, `_two`, `_few`, `_many`, `_other` |
| ru-RU | 3 | `_one`, `_few`, `_many` |

```njk
{{ i18n.plural('item', count) }}
```

```json5
// English
{ "item_one": "1 item", "item_other": "{{count}} items" }

// Indonesian (no plural)
{ "item_other": "{{count}} item" }

// Arabic (complex)
{
  "item_zero": "٠ عنصر",
  "item_one": "عنصر واحد",
  "item_two": "عنصران",
  "item_few": "{{count}} عناصر",
  "item_many": "{{count}} عنصر",
  "item_other": "{{count}} عنصر"
}
```

### RTL Support

Arabic (`ar-SA`) uses RTL automatically:

```njk
<html dir="rtl">  <!-- Set automatically for ar-SA -->
```

```css
/* ✅ Use logical properties (RTL-aware) */
.sidebar { margin-inline-start: 1rem; }

/* ❌ Avoid physical properties (not RTL-aware) */
.sidebar { margin-left: 1rem; }
```

### Regional Pricing

```json5
{
  "pricing": {
    "base": 29,           // Fallback: $29 USD
    "id-ID": 500000,      // Indonesia: Rp500.000
    "ja-JP": 4500,        // Japan: ¥4,500
    "ar-SA": 108.75       // Saudi: 108.75 SAR
  }
}
```

```njk
{{ i18n.formatLocalPrice(plan) }}
// id-ID → Rp500.000 (override)
// ja-JP → ¥4,500 (override)
// en-US → $29.00 (base)
// es-ES → €24.90 (base + exchange rate)
```

**Update rates:** `bun run fetch:rates`

### Locale-Specific Data

Key suffix patterns for regional values:

| Suffix | Purpose | Example |
|--------|---------|---------|
| `_label` | Display label | `email_label`, `phone_label` |
| `_value` | Actual value (locale-specific) | `email_value`, `phone_value` |
| `_val` | Numeric stat (locale-specific) | `years_val`, `clients_val` |

```njk
// Config: { "label": "email" }
{{ i18n.t('page.' + item.label + '_label') }}   // Email
{{ i18n.t('page.' + item.label + '_value') }}   // hello@starter.app
```

| Key | id-ID | ja-JP | ar-SA |
|-----|-------|-------|-------|
| `phone_value` | `+62 812 3456 7890` | `+81 90 1234 5678` | `+966 50 123 4567` |
| `location_value` | `Jakarta, Indonesia` | `東京，日本` | `الرياض، السعودية` |

## Best Practices

### ✅ DO

```njk
{{ i18n.t('page.hero.title') }}        # Use prefixes
{{ i18n.t('comp.cta.heading') }}      # Be specific
{{ i18n.html('page.content') }}        # Use html() for HTML
{{ i18n.formatCurrency(price) }}       # Format numbers
```

```css
.content { margin-inline-start: 1rem; }  # Logical for RTL
```

```json5
{
  "copyright": "© 2026",    # Unicode in JSON5
  "price": "99.99"          # Universal digits
}
```

### ❌ DON'T

```njk
<button>Submit</button>              # Don't hardcode
{{ i18n.t('hero.title') }}           # Don't omit prefix
{{ i18n.t('page.content') }}         # Don't use t() for HTML
```

```css
.sidebar { margin-left: 1rem; }      # Not RTL-aware
```

```json5
{
  "copyright": "&copy; 2026",        # Don't use HTML entities
  "price": "٩٩.٩٩"                   # Don't use native digits
}
```

## Common Pitfalls

| Problem | Solution |
|---------|----------|
| `[page.key.name]` in output | Add key to ALL 12 locale files |
| `&copy;` shows as text | Use `©` directly in JSON5 |
| Text not updating on switch | Use `i18n.html()` not `t()` for dynamic content |
| Variables not replacing | Check `{{var}}` syntax, pass object in template |
| RTL layout broken | Use `margin-inline-start` not `margin-left` |
| Currency wrong format | Use `formatCurrency()` not raw string |
